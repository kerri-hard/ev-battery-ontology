# 점진적 자율성의 다섯 단계 — L0에서 L4까지

> 📘 **자율 제조 사상 시리즈** — 3편 (총 5편)
> 전체 시리즈 목차는 글 하단에 있습니다.

---

## 0. 자율주행이 가르쳐 준 것

2014년 SAE International이 자율주행 단계(J3016)를 발표했을 때, 업계는 처음으로 *"자율"이 한 가지가 아니라는 사실*을 받아들였다. 어시스트(L1)와 완전 자율(L5) 사이에는 *세 단계의 신뢰 빌딩*이 있다.

| 자율주행 단계 | 핵심 |
|---|---|
| L0 | 사람이 모든 운전 |
| L1 | 시스템이 1가지 보조 (크루즈/차선 유지 중 하나) |
| L2 | 시스템이 2가지 동시 보조. 사람이 항시 감시 |
| L3 | 정해진 조건에서 시스템 자율. 사람은 *대기* |
| L4 | 정해진 영역에서 완전 자율. 사람 개입 없음 |
| L5 | 모든 조건 완전 자율 |

이 단계 분류가 산업계에 준 가장 큰 선물은, *현재 어디에 있는지 측정 가능*하게 만든 것이다. "자율주행 자동차"는 모호하지만, "L3 자율"은 구체적이다.

자율 제조도 같은 단계 분류가 필요하다.

---

## 1. 다섯 단계의 자율 제조

자율주행의 6단계(L0-L5)를 제조 도메인에 맞게 다섯 단계로 매핑한다.

| 단계 | 이름 | 사람의 역할 | 시스템의 역할 |
|---|---|---|---|
| **L0** | 수동 | 모든 결정 + 모든 실행 | 데이터 수집만 (대시보드) |
| **L1** | 권고 | 결정 + 실행 | 추천안 제시 |
| **L2** | 반자율 | HITL 승인 + 검토 | 액션 제안 + 실행 |
| **L3** | 자동 | 예외 검토 (사후) | 99% 자동 복구 |
| **L4** | 완전 자율 | 거버넌스/정책 | 전권 + 자가 진화 |

각 단계가 *어떤 결정에 적용되는지* 가 핵심이다. 한 시스템 안에서도 결정마다 단계가 다를 수 있다 — 안전 등급 A 공정은 L2까지만, C 등급 공정은 L4까지.

---

## 2. L0 — 수동 (모든 것은 사람)

### 무엇

데이터는 모이고 차트는 그려지지만, 결정은 모두 사람. 알람이 울리면 운영자가 매뉴얼을 보고 판단한다.

### 사람의 일

- 알람 받기
- 매뉴얼 따라 진단
- 액션 실행
- 사후 보고서

### 시스템의 일

- 센서 데이터 수집
- 대시보드/차트 표시
- 알람 룰 평가

### 한계

대부분의 *"스마트 팩토리"* 가 사실 L0다. 시스템이 *예쁜 정보*를 주지만, *결정*은 사람이다. 사람이 휴가 가면 라인이 멈춘다.

---

## 3. L1 — 권고 (시스템이 의견을 낸다)

### 무엇

시스템이 *추천안*을 제시한다. 사람이 그 추천을 받아들이거나, 무시하거나, 다른 결정을 내린다. 시스템의 결정 권한은 0.

### 사람의 일

- 추천 검토
- 받아들일지 결정
- 액션 실행

### 시스템의 일

- 진단 수행
- 회복 액션 추천 (top-3 후보)
- *왜 이 추천인지* 설명

### 신뢰의 수단

- 추천 정확도 통계 (운영자가 받아들인 비율 vs 거절한 비율)
- 추천의 *근거 그래프* — 어떤 인과 사슬에 기반했는지

### 함정

L1에 너무 오래 머무르면 운영자가 *추천을 의식 없이 받아들이는* 경향이 생긴다. 그러면 사실상 L2지만 책임은 사람에게 — 최악의 조합. L1은 시간 제한이 있어야 한다.

---

## 4. L2 — 반자율 (시스템이 실행, 사람이 승인)

### 무엇

시스템이 액션을 *준비*하고, 사람이 *승인*하면 실행한다. 사람의 승인은 빠른 클릭일 수 있지만 *결정 책임은 사람에게 있다*.

### 사람의 일

- 승인 큐 모니터링
- 승인/거절 클릭
- 거절한 경우 *왜* 거절했는지 피드백

### 시스템의 일

- 진단 + 액션 준비 + 시뮬레이션
- 사람의 결정 시간 측정 (병목 측정)
- 사람의 거절 사유 학습

### 신뢰의 수단

- 승인율: 99% 이상 → L3 후보
- 거절 사유의 패턴화: 사람이 어떤 근거로 거절하는지 학습
- 결정 시간: 너무 빠르면 의식 없이 누름 (L2의 함정)

### 핵심 — *Human-in-the-loop (HITL)* 의 진짜 위치

L2의 본질은 사람을 시스템 *안에* 둔 게 아니라, *결정 게이트*에 둔 것이다. 시스템은 모든 준비를 하고, 사람은 *go/no-go* 만 한다. 사람이 매뉴얼을 펼치고 분석하지 않는다 — 이미 시스템이 다 했다.

이 차이를 모르면 L2는 그냥 L1의 변형이 된다. 액션 준비를 사람이 또 하면 자율성 0.

---

## 5. L3 — 자동 (시스템이 결정, 사람이 사후 검토)

### 무엇

대부분의 결정을 시스템이 *자동으로* 한다. 사람은 사후 보고서나 예외 케이스만 검토한다.

### 사람의 일

- 일일 사후 검토 (어떤 결정이 있었나)
- 예외 케이스 검토 (자동 복구 실패한 것)
- 시스템의 *학습 결과* 검토 (어떤 새 규칙이 추가됐나)

### 시스템의 일

- 99% 자동 진단 + 결정 + 실행
- 신뢰도 낮으면 자동으로 L2로 강등 (HITL 호출)
- 학습 레코드 자동 영속화

### 신뢰의 수단

- 자동 복구 성공률 (auto_recovery_rate ≥ 95%)
- 캘리브레이션 (예측 신뢰도 vs 실제 정확도)
- 재발률 (같은 시그니처 incident가 반복되지 않음)

### L3의 분기

L3에서 한 가지 결정에 대해 시스템이 자동으로 두 가지 길로 갈 수 있다:

- **확신** → 자동 실행 + 사후 보고
- **불확신** → 자동으로 L2로 강등 → HITL 게이트

이 *동적 강등*이 L3를 안전하게 만든다. 항상 L3로 머물지 않고, 어떤 결정에 대해서는 L2로 자동 후퇴한다. 이를 *graceful degradation* 이라 부른다.

### L3의 위험

L3에서 운영자가 *너무 적게 본다*. 잘못된 자동 결정이 누적되어도 6개월 후에야 발견될 수 있다. 그래서 L3는 *사후 검토 의무*를 시스템 자체가 운영자에게 강제한다.

- 매주 자동 결정 샘플 무작위 추출 → 사람 리뷰
- 새로 발견된 패턴 → 사람 검증 후 영속화
- 신뢰도 캘리브레이션 변화 → 자동 알람

---

## 6. L4 — 완전 자율 (사람은 정책만)

### 무엇

시스템이 *전권*으로 운영한다. 사람은 *정책*만 정의한다.

### 사람의 일

- 시스템의 *목적 함수* 정의 (예: "수율 0.99 이상 + HITL률 0.05 이하 + 안전 등급 A 위반 0")
- 분기마다 정책 검토
- 거버넌스 (감사, 규제, 윤리)

### 시스템의 일

- 자율 운영
- *자가 진화* — 자기 임계값, 자기 진단 규칙, 자기 액션 카탈로그를 자가 조정
- 새로운 인과 자동 발견
- 학습 결과 사람 정책에 *제안* (수동 승인이 아니라 *제안*만)

### 신뢰의 수단

- 1년 이상 L3에서 무사고 운영
- 여러 번의 점진적 자율성 승급 검증
- 외부 감사 (제3자가 시스템 결정 로그를 *역추적* 가능해야 함)

### L4의 핵심 — *자가 진화*

L4 시스템은 자기 자신을 변형한다.

- 새 진단 규칙을 발견하고 검증해서 자기 그래프에 추가
- 회복 액션의 우선순위를 fitness EMA로 자가 조정
- 시뮬레이션 임계값을 예측 정확도에 따라 자가 보정

이 자가 진화가 사람의 검증 없이 진행된다는 점이 L4의 *무서움*이다. 그래서 L4는 *roll-back 메커니즘*이 강해야 한다 — 자가 진화가 메트릭을 악화시키면 즉시 자동 롤백.

### L4의 함정 — *해석 가능성 부채*

자가 진화가 누적되면 시스템이 *왜 그렇게 동작하는지* 설명하기 어려워진다. 그래서 L4 시스템은 다음을 강제한다.

- 모든 자가 진화 결정에 *근거*가 그래프로 영속화
- 6개월에 한 번씩 사람이 *진화 로그* 검토
- 해석 불가능한 결정은 자동 거절 (블랙박스 학습 금지)

## 7. 한 시스템 안에서의 단계 혼재

다크 팩토리는 *한 단계*에 머물지 않는다. 결정마다, 공정마다, 안전 등급마다 단계가 다르다.

| 결정 종류 | 일반 안전 | 안전 등급 A |
|---|---|---|
| 알람 평가 | L4 (완전 자율) | L4 |
| 진단 수행 | L4 | L3 |
| 회복 액션 추천 | L3 | L2 |
| 회복 액션 실행 | L3 | L2 (항상 HITL) |
| 정책 변경 | L2 | L0 (사람만) |
| 자가 진화 (임계값 등) | L4 | L1 (제안만) |

이 *세분화*가 자율 시스템의 핵심이다. "이 시스템은 L3" 라는 말은 너무 거칠다. "이 결정은 L3, 저 결정은 L2" 가 정확한 표현.

## 8. 단계 사이의 *승급*

L1에서 L2로, L2에서 L3로, L3에서 L4로 가는 것은 *기능 추가*가 아니라 *신뢰 증명*이다.

### 승급 조건의 예 (L2 → L3)

- 최근 30일간 사람의 승인율 ≥ 99%
- 거절 사유의 80% 이상이 시스템에 학습됨
- 캘리브레이션 (예측 신뢰도 vs 실제) 오차 ≤ 임계
- 결정 시간 패턴이 *의식적 검토*를 시사 (너무 빠르지 않음)

### 강등 조건의 예 (L3 → L2)

- 자동 복구 성공률 7일 평균이 임계 미달
- 같은 시그니처 incident가 반복 발생 (anti-recurrence 트리거)
- 시뮬레이션과 실제 결과 사이 캘리브레이션 악화
- 외부 환경 변화 감지 (새 시즌, 새 자재, 새 공정)

승급은 *조용히* 일어나지 않는다. 운영자에게 명시적으로 알림이 가고, 일정 기간 *유예 기간*을 둔다. 강등은 즉시.

## 9. 단계 분류가 경영에 주는 가치

### 9.1 ROI 계산이 명확해진다

| 단계 | 사람 인건비 절감 | 신규 도구 비용 |
|---|---|---|
| L0 → L1 | 0% (의사결정 그대로) | 추천 모델 도입 |
| L1 → L2 | 30% (분석 시간 줄어듦) | HITL 워크플로 |
| L2 → L3 | 70% (대부분 자동) | 캘리브레이션/학습 인프라 |
| L3 → L4 | 95% | 자가 진화 + 거버넌스 |

각 단계의 가치를 *분리해서* 측정 가능. "AI 도입 ROI"가 모호한 것과 정반대.

### 9.2 위험 관리가 명확해진다

규제 당국이나 보험사에게 *"우리는 L3"* 라고 말할 수 있다. 그게 의미하는 것이 분명하므로.

### 9.3 점진적 도입이 가능해진다

큰 시스템 전체를 한 번에 L4로 보내려는 시도는 거의 항상 실패한다. 결정 종류별로 단계를 다르게 적용하면, *낮은 위험 결정부터* 자율성을 늘릴 수 있다.

---

## 10. 마무리 — 자율은 *권리*가 아니라 *증명*이다

자율 시스템에 대한 가장 위험한 오해는, 자율성이 *시스템의 권리*라고 생각하는 것이다. 시스템이 똑똑해 보이면 자율성을 줘야 한다는 식으로.

자율성은 *증명*이다. 시스템이 L1에서 L2로 가려면, *사람이 받아들일 만큼* 정확해야 한다. L2에서 L3로 가려면, *캘리브레이션이 정직*해야 한다. L3에서 L4로 가려면, *자가 진화가 메트릭을 개선*해야 한다.

이 증명 과정이 누락되면, 시스템은 어느 단계에 있든 *블랙박스*가 된다. 그리고 첫 사고에서 모든 자율성이 회수된다.

> 점진적 자율성은 *느린 길*이 아니라 *유일한 길*이다.
