# 📘 블로그 에세이 — 자율 제조 사상 시리즈

이 디렉토리는 **다크 팩토리·자율 제조 사상**을 코드 디테일 없이 *이론과 용어* 중심으로 정리한 블로그용 글 시리즈입니다.

> 사상은 도구보다 오래 산다. 본 시리즈는 *왜 이렇게 생각하는가* 만 다룹니다.
> 구현/코드 디테일은 저장소 안 `docs/handbook/` 참조.

---

## 누구를 위한 글인가

- **경영진/의사결정자** — 다크 팩토리 도입 의사결정의 *사상적 배경* 이 필요한 사람
- **운영자/SRE** — 자율 제조의 SLI/SLO 사상이 궁금한 사람
- **연구자/엔지니어** — 코드보다 *왜 이렇게 설계했는가* 가 궁금한 사람
- **블로그 독자** — 산업 5.0/AIOps/자율 시스템에 일반적 관심이 있는 사람

코드 디테일은 일절 없습니다. 파일 경로, 함수 이름, API 엔드포인트, 명령어 — 모두 제외.

## 어떻게 쓰나 (네이버 블로그 게시)

각 챕터 = 네이버 블로그 글 1편. 권장 게시 순서: `01` → `02` → ... → `05`.

### 복사·붙여넣기 절차

1. `NN-*.md` 파일 1개를 에디터에서 열기
2. **전체 선택 → 복사**
3. 네이버 블로그 글쓰기 화면에 **그대로 붙여넣기**
4. 네이버 에디터에서 다음을 *수동* 처리:
   - **제목** — 글 첫 줄의 `# 제목` 을 네이버 제목 입력란에 옮기기 (본문에서 삭제)
   - **태그** — 글 마지막 `📎 키워드` 섹션의 단어를 태그 입력란에 옮기기
   - **헤딩 서식** — `##`, `###` 으로 시작하는 줄을 네이버 "제목 1/2/3" 서식으로 변환
   - **표** — `| ... |` 가 raw로 보이면 네이버의 표 위젯으로 다시 그리기
   - **인용 박스** — `>` 로 시작하는 줄을 네이버 "인용" 서식으로
   - **코드 블록** — ` ``` ` 사이의 줄을 네이버 "코드 블록" 위젯으로 (드물게 출현)

### 시리즈 연결

각 글 푸터에 시리즈 인덱스 5편이 자동 포함됩니다. 네이버에서는 다른 편으로
바로 점프 못 하므로, 글 마지막에 본인의 다른 글 URL을 수동 추가해 시리즈를
체이닝할 수 있습니다.

## 글의 톤 / 표기 규약

- **존대 ↔ 평어 혼합** — 글의 흐름에 따라. 사상적 단언은 평어, 독자에게 말 거는 부분은 자연스럽게.
- **굵은 영문 약어** — 첫 등장 시 굵게 (RCA, FMEA, SLO 등). 익숙해지면 일반 텍스트.
- **외래어** — 정착된 것은 한글 (자율 시스템, 자가 학습), 미정착은 영문 (Granger 인과성, error budget, anti-recurrence)
- **긴 인용** — `>` 인용 박스 + 전후 줄 띄우기
- **이모지** — 절제. 문단 강조용으로 1-2개만 (📘 📎 등)

## 코드 디테일이 *왜* 빠졌는가

이 시리즈의 의도된 제약:

- 본 저장소의 파일 경로/함수/API/명령어 인용 X
- 특정 라이브러리/프레임워크 이름 X (Python, FastAPI, Next.js 같은 도구)
- *왜* 그렇게 설계했는가의 *사상* 만 다룸

이유: 사상은 도구보다 오래 산다. 본 저장소가 5년 후 다른 언어/프레임워크로 재구현되어도, 이 시리즈의 글은 여전히 유효해야 한다.

코드 디테일이 필요한 독자는 저장소 안 `docs/handbook/` 을 참조.

## 후속 시리즈 후보 (현재 미작성)

이 5편으로 끝이 아닐 수 있습니다. 후속으로 다룰 만한 주제:

- **Hybrid Agentic 깊게** — 빠른 규칙 vs LLM 분기의 정량적 결정
- **자가 진화의 윤리** — 시스템이 자기 자신을 변형할 때의 거버넌스
- **운영자 워크벤치 UX** — *덜 보게 하는* 인터페이스 설계
- **제조업 vs IT의 자율성** — SRE 사상이 옮겨오면서 잃는 것 / 얻는 것

이 중 작성 우선순위는 운영 데이터와 독자 반응에 따라 결정.
